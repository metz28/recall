REINFORCEMENT LEARNING: FROM BASICS TO ADVANCED

Introduction

Reinforcement Learning (RL) is a type of machine learning where an agent learns to make decisions by interacting with an environment. The agent receives rewards or penalties for its actions and learns to maximize cumulative reward over time. Unlike supervised learning, there are no labeled examples - the agent must discover good strategies through trial and error.

Core Concepts

1. Agent
   - The learner or decision maker
   - Observes the environment
   - Takes actions
   - Learns from experience

2. Environment
   - The world the agent interacts with
   - Provides observations/states
   - Responds to actions
   - Gives rewards

3. State (s)
   - Current situation of the agent
   - All information needed to make decisions
   - Can be fully observable or partially observable

4. Action (a)
   - Choices available to the agent
   - Discrete: finite set (move left, right, up, down)
   - Continuous: infinite possibilities (steering angle, force)

5. Reward (r)
   - Immediate feedback from environment
   - Scalar value (positive or negative)
   - Guides learning process
   - Design is crucial for success

6. Policy (π)
   - Agent's strategy
   - Maps states to actions
   - π(a|s): probability of action a in state s
   - Goal: Find optimal policy π*

7. Value Function
   - Expected cumulative reward
   - State value V(s): Expected return from state s
   - Action value Q(s,a): Expected return from taking action a in state s
   - Helps evaluate policies

8. Return (G)
   - Total cumulative reward
   - Future rewards often discounted
   - G_t = r_t + γr_{t+1} + γ²r_{t+2} + ...
   - γ (gamma): discount factor (0 to 1)

The RL Problem

Markov Decision Process (MDP):
- Framework for RL problems
- Tuple: (S, A, P, R, γ)
  - S: State space
  - A: Action space
  - P: Transition probability P(s'|s,a)
  - R: Reward function R(s,a)
  - γ: Discount factor

Markov Property:
- Future depends only on current state
- Past history doesn't matter
- P(s_{t+1}|s_t, a_t, s_{t-1}, a_{t-1}, ...) = P(s_{t+1}|s_t, a_t)

Bellman Equations:
- Fundamental recursive relationships
- State value: V(s) = E[r + γV(s')]
- Action value: Q(s,a) = E[r + γmax_a'Q(s',a')]
- Basis for many RL algorithms

Exploration vs. Exploitation

Key Dilemma:
- Exploration: Try new actions to discover better strategies
- Exploitation: Use known good actions to maximize reward
- Must balance both for effective learning

Strategies:

1. ε-Greedy
   - Choose random action with probability ε
   - Choose best known action with probability 1-ε
   - Simple and effective
   - ε can decay over time

2. Upper Confidence Bound (UCB)
   - Add exploration bonus to value estimates
   - Explore less-tried actions
   - Principled approach

3. Thompson Sampling
   - Bayesian approach
   - Sample from probability distribution
   - Effective in bandit problems

4. Entropy Regularization
   - Add entropy term to objective
   - Encourages policy diversity
   - Used in modern deep RL

Traditional RL Algorithms

1. Dynamic Programming

   Value Iteration:
   - Iteratively update value function
   - V_{k+1}(s) = max_a E[r + γV_k(s')]
   - Converges to optimal value
   - Requires model of environment

   Policy Iteration:
   - Alternate between:
     1. Policy Evaluation: Compute V^π
     2. Policy Improvement: π' = greedy(V^π)
   - Converges to optimal policy
   - Often faster than value iteration

2. Monte Carlo Methods

   - Learn from complete episodes
   - No bootstrapping
   - Average actual returns
   - Model-free (no environment model needed)

   First-Visit MC:
   - Average returns from first visit to each state
   - Unbiased estimate

   Every-Visit MC:
   - Average returns from all visits
   - More data, slightly biased

3. Temporal Difference (TD) Learning

   - Learn from incomplete episodes
   - Bootstrap from current estimates
   - Combines MC and DP ideas

   TD(0):
   - V(s) ← V(s) + α[r + γV(s') - V(s)]
   - Update immediately after each step
   - More efficient than MC

   SARSA (State-Action-Reward-State-Action):
   - On-policy TD control
   - Q(s,a) ← Q(s,a) + α[r + γQ(s',a') - Q(s,a)]
   - Safe learning (follows current policy)

   Q-Learning:
   - Off-policy TD control
   - Q(s,a) ← Q(s,a) + α[r + γmax_a'Q(s',a') - Q(s,a)]
   - Learns optimal policy regardless of behavior
   - More data-efficient

Deep Reinforcement Learning

Motivation:
- Traditional RL: Tabular methods
- Problem: State/action spaces too large
- Solution: Function approximation with neural networks

1. Deep Q-Networks (DQN)

   Key Innovations (2015):
   - Use CNN to approximate Q-function
   - Experience Replay: Store transitions, sample randomly
   - Target Network: Separate network for stability

   Algorithm:
   1. Observe state s
   2. Select action: a = argmax_a Q(s,a) (with ε-greedy)
   3. Execute action, observe r, s'
   4. Store transition (s,a,r,s') in replay buffer
   5. Sample mini-batch from buffer
   6. Update Q-network:
      y = r + γmax_a'Q_target(s',a')
      Loss = (y - Q(s,a))²
   7. Periodically update target network

   Improvements:
   - Double DQN: Reduce overestimation
   - Dueling DQN: Separate value and advantage
   - Prioritized Experience Replay: Sample important transitions
   - Rainbow: Combines multiple improvements

2. Policy Gradient Methods

   Idea: Directly optimize policy
   - Parameterize policy: π_θ(a|s)
   - Maximize expected return: J(θ)
   - Update using gradient ascent

   REINFORCE:
   - Monte Carlo policy gradient
   - θ ← θ + α∇log(π_θ(a|s))G_t
   - High variance

   Actor-Critic:
   - Actor: Policy network
   - Critic: Value network
   - Reduce variance with baseline
   - θ ← θ + α∇log(π_θ(a|s))[G_t - V(s)]

3. Advanced Policy Gradient Algorithms

   A3C (Asynchronous Advantage Actor-Critic):
   - Multiple parallel agents
   - Share global network
   - Asynchronous updates
   - Stabilizes training

   PPO (Proximal Policy Optimization):
   - Clip objective to prevent large updates
   - Simple and effective
   - Most popular algorithm
   - Used in ChatGPT (RLHF)

   TRPO (Trust Region Policy Optimization):
   - Constrain policy update size
   - Monotonic improvement guarantee
   - Complex implementation

   SAC (Soft Actor-Critic):
   - Off-policy algorithm
   - Entropy regularization
   - Maximum entropy RL
   - Sample efficient

4. Continuous Control

   DDPG (Deep Deterministic Policy Gradient):
   - Actor-critic for continuous actions
   - Deterministic policy
   - Off-policy learning
   - Use experience replay

   TD3 (Twin Delayed DDPG):
   - Two critic networks
   - Delayed policy updates
   - Target policy smoothing
   - Reduces overestimation

   SAC (mentioned above):
   - State-of-the-art continuous control
   - Stochastic policy
   - Automatic temperature tuning

5. Model-Based RL

   Learn environment model:
   - Predict next state and reward
   - Plan using learned model
   - More sample efficient

   Approaches:
   - Dyna-Q: Integrate planning and learning
   - World Models: Learn compressed representation
   - MuZero: Combines tree search and learning
   - Dreamer: Learn in latent space

Multi-Agent RL

Challenges:
- Non-stationary environment
- Credit assignment
- Partial observability
- Communication

Approaches:
- Independent Learners: Each agent learns separately
- Centralized Training, Decentralized Execution
- Communication protocols
- Cooperative vs. Competitive

Algorithms:
- MADDPG: Multi-agent DDPG
- QMIX: Value decomposition
- CommNet: Learning to communicate

Inverse Reinforcement Learning

Problem: Learn reward function from demonstrations
- Observe expert behavior
- Infer underlying reward
- Use learned reward to train agent

Applications:
- Imitation learning
- Apprenticeship learning
- Understanding human behavior

Hierarchical RL

Motivation: Complex tasks need temporal abstraction
- Options framework: Temporally extended actions
- Feudal networks: Hierarchy of policies
- Goal-conditioned RL: Subgoals

Benefits:
- Faster learning
- Better exploration
- Transfer learning

Meta-RL

Learning to learn:
- Train on distribution of tasks
- Adapt quickly to new tasks
- Few-shot RL

Approaches:
- MAML: Model-Agnostic Meta-Learning
- RL²: Fast RL via slow RL
- Meta-learning with memory

Challenges in RL

1. Sample Efficiency
   - Requires many environment interactions
   - Solutions: Off-policy, model-based, transfer learning

2. Reward Design
   - Difficult to specify good rewards
   - Reward shaping, inverse RL
   - Reward hacking

3. Stability
   - Training can be unstable
   - Careful hyperparameter tuning
   - Experience replay, target networks

4. Exploration
   - Hard exploration problems
   - Sparse rewards
   - Intrinsic motivation, curiosity

5. Partial Observability
   - Agent doesn't see full state
   - Use recurrent networks
   - Model uncertainty

6. Safety
   - Exploration can be dangerous
   - Constrained RL
   - Safe exploration strategies

Best Practices

1. Start Simple
   - Test on simple environments first
   - CartPole, MountainCar
   - Verify implementation

2. Hyperparameter Tuning
   - Learning rate most critical
   - Use proven configurations
   - Systematic search

3. Monitor Training
   - Plot reward curves
   - Track other metrics
   - Use TensorBoard

4. Reproducibility
   - Set random seeds
   - Document hyperparameters
   - Version environments

5. Debugging
   - Check environment is working
   - Test with random policy
   - Simplify problem

6. Evaluation
   - Separate train/test environments
   - Multiple seeds
   - Disable exploration during evaluation

7. Reward Engineering
   - Start with dense rewards
   - Avoid unintended behaviors
   - Reward shaping carefully

Conclusion

Reinforcement Learning is a powerful paradigm for sequential decision-making. From game-playing to robotics, RL has achieved remarkable successes. Deep RL combines the flexibility of neural networks with the RL framework, enabling agents to solve complex tasks. While challenges remain in sample efficiency, stability, and safety, ongoing research continues to push the boundaries of what's possible with RL.
